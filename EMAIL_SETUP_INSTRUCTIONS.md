# 📧 Инструкция по настройке Gmail для пересылки писем

## ⚠️ Проблема
При включенной **двухфакторной аутентификации (2FA)** в Gmail обычный пароль **не работает** для доступа через IMAP/SMTP. Нужен специальный **App Password** (пароль приложения).

## ✅ Решение: Создание App Password в Gmail

### Шаг 1: Убедитесь, что 2FA включена
1. Перейдите на https://myaccount.google.com/security
2. В разделе "Вход в аккаунт Google" найдите **"Двухэтапная аутентификация"**
3. Если не включена — включите её (это обязательно!)

### Шаг 2: Создайте App Password (пароль приложения)
1. Перейдите на **https://myaccount.google.com/apppasswords**
   - Или: Настройки Google → Безопасность → **Пароли приложений**
2. Введите пароль от аккаунта Google (если потребуется)
3. Выберите приложение: **"Почта"**
4. Выберите устройство: **"Другое (произвольное имя)"**
5. Введите имя: **"Helpdesk System"** (или любое другое)
6. Нажмите **"Создать"**
7. **⚠️ ВАЖНО: Скопируйте 16-значный пароль!** (он будет показан один раз!)
   - Пароль выглядит как: `abcd efgh ijkl mnop` (можно вводить с пробелами или без)

### Шаг 3: Обновите настройки в проекте

**Вариант 1: Через переменные окружения (РЕКОМЕНДУЕТСЯ)**
```bash
# В терминале перед запуском сервера:
export EMAIL_HOST_PASSWORD='ваш-16-значный-пароль-приложения'

# Или добавьте в ~/.zshrc или ~/.bashrc:
echo 'export EMAIL_HOST_PASSWORD="ваш-пароль"' >> ~/.zshrc
source ~/.zshrc
```

**Вариант 2: Прямо в settings.py (только для разработки!)**
Обновите строку в `backend/core/settings.py`:
```python
EMAIL_HOST_PASSWORD = os.environ.get('EMAIL_HOST_PASSWORD', 'ваш-16-значный-пароль-приложения')
```

### Шаг 4: Проверьте работу
```bash
cd backend
python3 manage.py check_email
```

Если всё настроено правильно, вы увидите:
- ✓ Подключено к почтовому серверу
- Найдено непрочитанных писем: X
- ✓ Переслано: [тема письма] → myrzabekerkebulanakt@gmail.com

## 📋 Что логируется

Все операции записываются в:
- **Консоль** (при запуске команды)
- **Файл:** `backend/logs/email_forwarding.log`

В логах вы увидите:
- Время подключения к почте
- Количество найденных писем
- Успешные пересылки
- Ошибки (если есть)

## ⚠️ Важно:
- App Password — это **16 символов** (может быть с пробелами или без)
- Этот пароль используется **вместо** обычного пароля
- Один App Password можно использовать для всех приложений
- Если пароль не работает — создайте новый App Password
- **НЕ коммитьте пароль в git!** Используйте переменные окружения

## 🔒 Безопасность:
- ✅ Используйте переменные окружения (`export EMAIL_HOST_PASSWORD=...`)
- ✅ Добавьте `.env` в `.gitignore` (уже добавлено)
- ❌ НЕ храните пароли в коде!

## 🐛 Если не работает:
1. Проверьте, что 2FA включена
2. Убедитесь, что используете App Password, а не обычный пароль
3. Проверьте логи: `cat backend/logs/email_forwarding.log`
4. Попробуйте создать новый App Password
